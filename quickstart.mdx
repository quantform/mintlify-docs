It's is an event-driven, composable runtime that models markets and protocols as deterministic streams and executes reactive backtesting-to-live strategy workflows through reusable, environment-agnostic components.

## Your strategy, your control

Quantform Core follows a developer-controlled model:

- **You own the logic** - Strategy decisions are defined by your code
- **You choose the mode** - Run in `paper`, `replay`, `live`, or `idle`
- **You inspect behavior** - Use logging and deterministic replay windows
- **You keep one codebase** - Promote the same strategy from test to production

## What this means in practice

Quantform turns market and protocol activity into event streams, so your automation can react to data in real time with deterministic behavior in testing.

- **Works across venues and protocols** - connect centralized exchanges, DeFi protocols, and wallets in one system.
- **Real-time by default** - react when market state changes, not through slow polling loops.
- **One strategy, multiple environments** - run the same strategy in replay, paper, and live modes.
- **Built for full automation** - automate more than orders, including rebalancing and execution workflows.

<Warning>
  Quantform Core is an execution framework, not a profit guarantee. Always test in `paper` and `replay` modes before running any live strategy.
</Warning>

## What you can build

With Quantform Core, you can build:

- Systematic trading strategies for spot, derivatives, and multi-venue setups
- Cross-market execution that routes orders based on spread, depth, or fees
- DeFi automation like position management and trigger-based operations
- Monitoring and response pipelines that detect events and execute actions

## How it works

1. Connect data sources (prices, orderbooks, protocol events).
2. Compose strategy logic as reactive pipelines.
3. Attach execution adapters to place orders or transactions.
4. Run in replay or paper mode for validation, then promote to live.
5. Reuse the same strategy core while swapping runtime dependencies.

## Simple mental model

- **Strategy code** = your rules
- **Runtime** = the engine executing those rules
- **Adapters/connectors** = data + execution bridges
- **Modes** = testing path (`replay`/`paper`) and production path (`live`)

## What Quantform Core is not

- Not an exchange
- Not a profit guarantee
- Not a replacement for risk management
- Not limited to one venue or one strategy style
